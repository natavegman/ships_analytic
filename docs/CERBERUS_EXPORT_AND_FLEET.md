# Цербер (cerberus.vetrf.ru) — реестр экспортёров и судов

## Что это

**Цербер** — публичный реестр Россельхознадзора ([cerberus.vetrf.ru/cerberus/certified/pub](https://cerberus.vetrf.ru/cerberus/certified/pub)): предприятия и объекты, **сертифицированные для экспорта** подконтрольной продукции (в т.ч. рыба и морепродукты) в страны назначения.

## Почему полезно для Quotas_analytic

1. **Связка компания ↔ страны экспорта**  
   Фильтр «Страна»: Корея, Китай, США, Европа и т.д. — видно, кто сертифицирован на вывоз в конкретную страну.

2. **ИНН/ОГРН**  
   В реестре есть «ИНН/ОГРН хоз. субъекта» — можно стыковать с `company_groups_enriched.csv` по ИНН.

3. **Суда в реестре**  
   Вид объекта: **«предприятия (суда) по добыче, переработке и транспортировке гидробионтов»** — даёт объекты, которые являются судами (название объекта = название судна или судовое предприятие). Отдельно есть «предприятия по добыче, переработке и транспортировке гидробионтов» (без слова «суда») — береговые/переработка.

4. **Выгрузка XLS**  
   На странице есть кнопка **«Сформировать новый отчет в формате xls»** — в диалоге можно задать количество записей, смещение и список полей. Скачивание по ссылке вида `/cerberus/reports/download?id=...`.

## Основные фильтры (на сайте)

| Фильтр | Примеры значений |
|--------|-------------------|
| Поднадзорный объект / Вид объекта | предприятия **(суда)** по добыче, переработке и транспортировке гидробионтов |
| ИНН/ОГРН хоз. субъекта | 10 или 12 цифр |
| Страна | Корея, Республика; Китай; Соединенные Штаты Америки; страны ЕС |
| Регион | Сахалинская обл., Приморский край, и т.д. |
| Тип продукции | **Рыба и морепродукты** |
| Виды деятельности | вылов, переработка, хранение, реализация и т.д. |

## Поля отчёта (для выгрузки)

Из диалога «Сформировать отчет» можно включить в XLS, в частности:

- № в реестре  
- Название объекта (для судов — название судна/объекта)  
- Вид объекта  
- Хоз. субъект, осуществляющий деятельность (наименование)  
- Хоз. субъект … (ИНН), (ОГРН)  
- Хоз. субъект - владелец  
- Регион, Адрес  
- Продукция, Виды деятельности  
- Статус, Статус аттестованной деятельности  

## Ограничения

- Сайт — SPA, после загрузки данные подгружаются по запросу (поиск, отчёт). Прямого публичного API в открытом доступе нет; для автоматизации нужна имитация браузера (Playwright/Selenium).
- Выгрузка XLS, скорее всего, с пагинацией/лимитом (например, не более N записей за один отчёт). Для полной базы может понадобиться несколько выгрузок (по странам, по виду объекта и т.д.).
- Ветис.API (упоминается в справке) предназначен для интеграции и, как правило, требует регистрации/доступа.

## Автоматическое скачивание (без вашего участия)

Скрипт **`scripts/cerberus_download_auto.py`** открывает сайт в головном браузере (Playwright), выставляет фильтр «Рыба и морепродукты», нажимает «Поиск», формирует отчёт XLS и сохраняет его в `data/cerberus_export_latest.xlsx`, затем парсит и обновляет `data/cerberus_export.csv` и `output/companies_with_export.csv`.

**Установка (один раз):**
```bash
pip install playwright pandas openpyxl
playwright install chromium
```

**Запуск:**
```bash
cd /path/to/Quotas_analytic
python3 scripts/cerberus_download_auto.py
```
С окном браузера (для отладки): `python3 scripts/cerberus_download_auto.py --headed`

**По расписанию (cron):** например раз в неделю:
```bash
5 9 * * 1 cd /Users/natalia/Code/Quotas_analytic && python3 scripts/cerberus_download_auto.py >> logs/cerberus.log 2>&1
```

Если Россельхознадзор изменит вёрстку или текст кнопок, в скрипте может понадобиться поправить селекторы (поиск по тексту «Поиск», «Сформировать новый отчет в формате xls» и т.д.).

## План использования

1. **Ручная выгрузка (быстрый старт)**  
   Зайти на [cerberus.vetrf.ru/cerberus/certified/pub](https://cerberus.vetrf.ru/cerberus/certified/pub) → Тип продукции: «Рыба и морепродукты» → Страна: по очереди Корея, Китай, США, нужные страны ЕС → Поиск → «Сформировать новый отчет в формате xls» → скачать XLS. Аналогично отдельно выгрузить объекты с видом «предприятия (суда) по добыче…» для списка судов/судопредприятий.

2. **Скрипт (автоматизация)**  
   В репозитории добавлен скелет скрипта `scripts/fetch_cerberus_export.py`, который рассчитан на работу через Playwright: открытие страницы, установка фильтров (продукция = рыба/морепродукты, страна, при необходимости вид объекта = суда), запуск отчёта XLS и сохранение файла. После скачивания XLS можно парсить и мержить с `company_groups_enriched.csv` по ИНН.

3. **Новая база**  
   - `data/cerberus_export.csv` — все выгруженные строки (ИНН, название объекта, вид объекта, страна, регион, продукция, виды деятельности, статус).  
   - Связка с квотами: по ИНН с `company_groups_enriched.csv` и при необходимости с `quota_summary.csv`.

4. **Суда**  
   Строки с видом объекта «предприятия (суда) по добыче, переработке и транспортировке гидробионтов» складывать в отдельную таблицу или помечать в `data/cerberus_export.csv` (например, колонка `is_vessel=1`). Название объекта в этих строках использовать как название судна/судового объекта для последующего сопоставления с РМРС/FleetPhoto при необходимости.

## РМРС и FleetPhoto (ограничения с ~2022)

Сведения о **судовладельце** в РМРС и на FleetPhoto в открытом доступе **скрыты примерно с 2022 года** в связи с санкциями. Поэтому по ним можно сопоставлять только по названию судна (как в `merge_rmrs_fleetphoto.py`). Альтернативные достоверные источники, где владелец/оператор может быть доступен (FAO Global Record, Global Fishing Watch API, ГСС/ИСРС), описаны в **[FLEET_SOURCES_ALTERNATIVES.md](FLEET_SOURCES_ALTERNATIVES.md)**.

### РМРС (Российский морской регистр судоходства)

- **Сайт:** [lk.rs-class.org](https://lk.rs-class.org) (личный кабинет), [rs-head.spb.ru](http://www.rs-head.spb.ru/)
- Реестр российских судов, классификационное общество. Публичного API нет; поиск по названию, IMO, порту приписки, судовладельцу — через веб-интерфейс (возможна регистрация).

### FleetPhoto (fleetphoto.ru)

- **Сайт:** [fleetphoto.ru](https://fleetphoto.ru) — галерея и база данных речных и морских судов.
- Список судов РМРС (с 1992): [fleetphoto.ru/list.php?lang=ru&rgid=2](https://fleetphoto.ru/list.php?lang=ru&rgid=2) — постранично, без API.
- На карточке судна есть: название, **проект/тип судна**, порт приписки, IMO, регистр, **Текущее состояние**; поля «судовладелец»/ИНН нет.
- **Статус:** сохраняется как на FleetPhoto — текст из поля «Текущее состояние» без перевода (Эксплуатируется, Прочее, Продан, Утилизирован и т.д.).
- **Траулеры и типы:** [fleetphoto.ru/projects/34/](https://fleetphoto.ru/projects/34/) — класс «Траулеры» с подразделами по проектам (1328 тип Балтика, 394 тип Маяковский и т.д.). В обогащении сохраняются тип/проект судна (`fleetphoto_project`) и прежние названия.

### Интеграция в проекте

1. **Колонки в `output/companies_with_export.csv`:**  
   **Цербер_судов** (из выгрузки Цербера по ИНН), **РМРС_судов**, **FleetPhoto_судов**.

2. **Скрипт `scripts/merge_rmrs_fleetphoto.py`:**
   - Скачивает страницы списка РМРС с FleetPhoto (list.php?rgid=2), кэш в `data/fleetphoto_rmrs_vessels.txt`.
   - Загружает суда из `data/cerberus_export.csv` (Судно=1).
   - Сопоставляет по нормализованному названию судна.
   - Для каждой компании записывает в **РМРС_судов** и **FleetPhoto_судов** количество судов из Цербера, найденных в списке FleetPhoto/РМРС.
   - Запуск: `python3 scripts/merge_rmrs_fleetphoto.py`

3. **Скрипт `scripts/enrich_fleetphoto_vessels.py`:** по судам из `gfw_our_vessels.json`, найденным в списке FleetPhoto, загружает карточку каждого судна и сохраняет: фото, статус (Текущее состояние), IMO, **тип/проект судна** (`fleetphoto_project`), старые названия. Запуск: `python3 scripts/enrich_fleetphoto_vessels.py`.

4. **Ограничения:** Совпадение только по названию; названия в Цербере и на FleetPhoto часто различаются. Для точного учёта по РМРС нужны данные из личного кабинета РМРС (судовладелец/ИНН).
