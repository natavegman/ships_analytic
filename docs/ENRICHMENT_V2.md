ТЗ: Рефакторинг модуля обогащения компаний (Enrichment Pipeline)
📌 1. Контекст и Цели (Для ИИ-ассистента)
Текущее состояние: Канонический скрипт `scripts/enrich_companies_to_db.py` берет ИНН компаний из legacy CSV, обогащает данные через DaData API и записывает результат напрямую в PostgreSQL (`companies`).
Проблемы: 1. Парсер периодически ловит капчу/блокировку, из-за чего в поле Юр_Лицо записывается мусор: "Проверка контрагента по ИНН – больше, чем выписка из ЕГРЮЛ". В базе сейчас 13 таких артефактов.
2. Система заточена только под компании с квотами. Нам нужно вносить в базу вручную финансовые хабы (банки, торговые дома, например АО "Гидрострой" или Банк "Итуруп"), и скрипт должен корректно их обогащать и присваивать им роли.
Целевое состояние: 1. Внедрить надежный API (DaData) для резолва ИНН -> Название компании (чтобы исключить парсинг HTML для базовых данных).
2. Написать функцию очистки (Data Cleansing) для существующих артефактов.
3. Расширить модель данных колонкой Роль_в_холдинге.

🛠 2. Стек технологий
Язык: Python 3.10+

Библиотеки: pandas, requests, dadata (официальная библиотека).

API: API DaData.ru (Подсказки по организациям) — бесплатный лимит 10 000 запросов/день, идеален для ИНН.

🚀 3. План реализации (Спринты для Cursor)
Спринт 1: Интеграция DaData API и Очистка
Задача: Заменить или дополнить нестабильный парсинг HTML надежным API-запросом.

Шаг 1.1: Зарегистрировать аккаунт на dadata.ru, получить API Key. Добавить чтение DADATA_API_KEY из .env.

Шаг 1.2: Написать функцию get_company_info_by_inn(inn: str) -> dict, которая делает запрос к DaData и возвращает чистое Юр_Лицо (полное или краткое), ОГРН, КПП, ФИО руководителя и статус (действует/ликвидирована).

Шаг 1.3: Написать утилиту-чистильщик clean_artifacts(df: pd.DataFrame) -> pd.DataFrame. Она должна найти все строки, где Юр_Лицо содержит "Проверка контрагента", взять ИНН этой строки, прогнать через get_company_info_by_inn и перезаписать правильное название.

Спринт 2: Расширение логики справочника (Поддержка хабов)
Задача: Научить скрипт работать с компаниями, добавленными вручную (без квот).

Шаг 2.1: Модифицировать входной файл company_groups.csv (или логику чтения). Добавить поддержку колонки Роль_в_холдинге (например: "Добыча", "Торговый дом", "Банк", "Строительство").

Шаг 2.2: Изменить логику основного цикла обогащения: скрипт должен брать ИНН, всегда сначала дергать надежный API DaData (для названия и директора), и только если нужны специфические связи (учредители-физлица), делать fallback-запрос на list-org/audit-it.

Шаг 2.3: Обработка ошибок: если компания в стадии ликвидации или присоединения (DaData отдает статус LIQUIDATING), скрипт должен писать это в колонку Комментарий (например: "В процессе присоединения").
