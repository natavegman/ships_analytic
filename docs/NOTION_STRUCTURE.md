# Структура Notion: Управление флотом и оборудованием

Архитектура рабочей базы для управления судами, оборудованием, заявками ЗИП и производственными показателями.

## Базы данных (Databases)

### 1. Суда (Vessels) — главная сущность

**Назначение:** Реестр всех судов. Центральный хаб, от которого строятся все связи.

| Свойство | Тип | Значения / Формат | Обязательное |
|----------|-----|-------------------|--------------|
| **Название судна** | Title | — | Да |
| **IMO** | Text | 7-значный номер | Нет |
| **Бортовой номер** | Text | АК-0445, МК-0271 и т.д. | Нет |
| **Состояние** | Select | `Эксплуатация` · `Ремонт` · `Отстой` | Да |
| **Тип/Модель (проект)** | Select | СРТМ, БМРТ, СТР, БАТМ, РС и т.д. | Нет |
| **Год постройки** | Number | 1985, 2020... | Нет |
| **Регион работы** | Select | `Северный` · `Дальневосточный` · `Норвежский` · `Прочий` | Нет |
| **Судовладелец** | Relation → Компании | — | Да |
| **GFW ID** | Text | UUID из Global Fishing Watch | Нет |
| **GFW Name** | Text | Латинское имя | Нет |
| **Фото** | Files & media | — | Нет |
| **Примечания** | Text | — | Нет |
| _Лебедки_ | Relation → Лебедки | (обратная связь) | — |
| _Спутниковые системы_ | Relation → Спутниковые системы | (обратная связь) | — |
| _Контроль трала_ | Relation → Контроль трала | (обратная связь) | — |
| _Доп. оборудование_ | Relation → Доп. оборудование | (обратная связь) | — |
| _Заявки ЗИП_ | Relation → Заявки ЗИП | (обратная связь) | — |
| _Вылов/Продукция_ | Relation → Вылов и продукция | (обратная связь) | — |
| **Кол-во лебедок** | Rollup | Count(Лебедки) | — |
| **Кол-во датчиков трала** | Rollup | Count(Контроль трала) | — |
| **Открытые заявки** | Rollup | Count(Заявки ЗИП where Статус ≠ Закрыта) | — |

**Views (представления):**

| View | Тип | Назначение |
|------|-----|------------|
| Все суда | Table | Полная таблица с фильтрами и сортировкой |
| По состоянию | Board (Kanban) | Группировка: Эксплуатация / Ремонт / Отстой |
| По регионам | Board | Группировка по региону работы |
| Карточка судна | Gallery | Визуальные карточки с фото и ключевыми параметрами |
| Мои суда | Table + фильтр | Фильтр по ответственному (если добавить поле Person) |

## Шаблоны (Templates)

### Шаблон карточки судна (в базе Суда)
При создании нового судна автоматически заполняются:
- Состояние = Эксплуатация
- Пустые секции для оборудования (инструкция заполнения в body)

### Шаблон заявки ЗИП (в базе Заявки ЗИП)
- Статус = Новая
- Приоритет = Средний
- Дата заявки = сегодня
- Чек-лист в body: ☐ Определить ЗИП → ☐ Запросить КП → ☐ Согласовать → ☐ Оплатить → ☐ Отследить отгрузку → ☐ Получить → ☐ Установить

### Шаблон запроса данных с судна (в базе Заявки ЗИП, тип = Диагностика)
- Чек-лист: ☐ Отправить запрос капитану → ☐ Получить ответ → ☐ Обновить данные в Notion

## Связи между базами (Relations & Rollups)

```
Суда ──────┬──► Компании (Судовладелец)
           ├──► Лебедки (1:N)
           ├──► Спутниковые системы (1:N)
           ├──► Контроль трала (1:N)
           ├──► Доп. оборудование (1:N)
           ├──► Заявки ЗИП (1:N)
           └──► Вылов/Продукция (1:N)

Компании ──► Квоты (1:N)

Лебедки ──────────► Заявки ЗИП (1:N)
```

## Порядок создания в Notion

1. **Компании** — создать первой (нет зависимостей)
2. **Суда** — добавить relation → Компании
3. **Квоты** — добавить relation → Компании
4. **Лебедки** — добавить relation → Суда
5. **Спутниковые системы** — добавить relation → Суда
6. **Контроль трала** — добавить relation → Суда
7. **Доп. оборудование** — добавить relation → Суда
8. **Вылов и продукция** — добавить relation → Суда
9. **Заявки ЗИП** — добавить relations → Суда, Лебедки, Спутн., Контроль, Доп.
10. **Дашборд** — linked views из всех баз

## Импорт данных

Для первичного заполнения подготовлены CSV-файлы (скрипт `scripts/prepare_notion_import.py`):

| CSV файл | Целевая база Notion |
|----------|---------------------|
| `notion_import/companies.csv` | Компании |
| `notion_import/vessels.csv` | Суда |
| `notion_import/quotas.csv` | Квоты |

Оборудование (лебедки, спутниковые системы, контроль трала, доп. оборудование) заполняется вручную или через API, т.к. этих данных в текущем проекте нет — они собираются на следующем этапе.

### Импорт CSV в Notion

1. Открыть целевую базу в Notion
2. `•••` → `Merge with CSV` (или создать новую базу `Import → CSV`)
3. Загрузить соответствующий CSV
4. Проверить маппинг колонок
5. После импорта настроить Relations вручную (Notion не импортирует связи из CSV)

### Автоматизация через Notion API

Для автоматического создания баз и импорта данных: `scripts/notion_create_databases.py`.
Требует `NOTION_API_TOKEN` в `.env`.
