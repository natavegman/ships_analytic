# CRAVT и GFW: обогащение базы судов и данные о выловах

Краткий обзор ресурсов [CRAVT](https://cravt.imcsnet.org/) (IMCS Network) и [Global Fishing Watch](https://globalfishingwatch.org/) применительно к нашей базе судов и запросу по выловам.

---

## 1. CRAVT (Combined Regional Vessel Authorisation Tool)

**Сайт:** [cravt.imcsnet.org](https://cravt.imcsnet.org/)  
**Описание:** [About the Data Sources](https://cravt.imcsnet.org/about-the-sources)

CRAVT объединяет реестры судов, **авторизованных для промысла** в зонах действия региональных организаций по управлению рыболовством (RFMOs/RFBs):

| Организация | Реестр | Регион / объект |
|-------------|--------|------------------|
| **CCSBT** | Record of Vessels (fishing + carrier) | Южный голубой тунец |
| **IOTC** | Record of Fishing Vessels | Индийский океан (IOTC) |
| **IATTC** | Regional Vessel Register | Восточная часть Тихого океана (тунец) |
| **WCPFC** | Record of Fishing Vessels | Западная и центральная часть Тихого океана |
| **NPFC** | Flagged Vessels Register | Северная часть Тихого океана (North Pacific) |
| **FFA** | FFA Vessel Register | Воды стран FFA, ВСМ в Тихом океане |
| **SPRFMO** | Record of Vessels | Южная часть Тихого океана (SPRFMO) |

Возможности CRAVT (по описанию на imcsnet.org):
- поиск судов по компании, **владельцу (owner)**, **оператору (operator)**;
- просмотр контактов судна (Vessel Contacts: компании и капитаны);
- сравнение записей по одному судну между разными RFMOs;
- просмотр по флагу, типу судна, организации.

### API / программный доступ CRAVT

На официальных страницах [CRAVT](https://cravt.imcsnet.org/about-the-sources) и [IMCS о CRAVT](https://imcsnet.org/combined-regional-authorised-vessels-tool) **ничего не сказано про API** — описан только веб-инструмент (поиск, просмотр по флагу/RFMO, Vessel Contacts). Утверждения в интернете про «CRAVT API для некоммерческого использования» могут относиться к другому сервису (например, к **GFW**, у которого действительно есть документированный бесплатный API для некоммерческого использования). Если нужен программный доступ или выгрузка данных CRAVT, имеет смысл уточнить напрямую: **mcs.network@imcsnet.org**.

При появлении API или выгрузок можно:
1. **Обогатить нашу базу судов:** по IMO / названию / флагу искать судно в CRAVT и подтягивать:
   - владелец (owner), оператор (operator);
   - контакты (Vessel Contacts — компании, капитаны);
   - в каких RFMOs судно авторизовано (WCPFC, NPFC и т.д.);
   - если в данных RFMOs есть фото — использовать их (зависит от формата API).
2. **Сопоставление с нашим списком:** у нас в `data/gfw_our_vessels.json` есть суда с ИНН, названием, при необходимости IMO; можно матчить по IMO или латинскому названию и дополнять поля `cravt_owner`, `cravt_operator`, `cravt_sources` (список RFMOs), при наличии — ссылку на фото.

### Ограничения

- CRAVT покрывает в основном **международные реестры** (тунец, тихоокеанские/индоокеанские зоны). Суда, работающие только в российских водах (Баренцево, Охотское и т.д.) и не внесённые в WCPFC/NPFC/др., в CRAVT могут отсутствовать.
- Фото в реестрах RFMOs есть не у всех судов; наличие полей в API нужно уточнять после получения доступа.

## 2. Global Fishing Watch (GFW)

**Портал API:** [globalfishingwatch.org/our-apis](https://globalfishingwatch.org/our-apis/documentation)

У GFW действительно есть **официальный API для некоммерческого использования** (регистрация, токен): Vessel API, Events API, 4Wings и др. — всё описано в документации на сайте.

Мы уже используем GFW:
- **Vessel API** — поиск судов по названию, идентификация (gfw_id, IMO, флаг, owner/operator из публичных реестров);
- в проекте: `scripts/gfw_client.py`, `scripts/build_gfw_vessel_cache.py`, `scripts/enrich_vessels_from_gfw.py` — кэш судов по нашим компаниям в `data/gfw_our_vessels.json`.

GFW даёт:
- **Активность и усилия (effort):** где и когда судно вело промысел (AIS/VMS), типы событий (fishing, port visits, encounters) — через Events API, 4Wings (карта).
- **Идентификацию судна:** название, IMO, флаг, владелец/оператор из открытых источников.

### Выловы (тонны улова)

**GFW не предоставляет данных о фактическом вылове (catch/landings) в тоннах.** В API есть только производные от трекинга: факт и зона активности, длительность, тип поведения — не объёмы добычи. Официальные выловы по судам/компаниям остаются в ведении государств и RFMOs (отчёты, статистика), не в GFW.

Итого: мониторинг «периодов тралений» (когда и где судно ловило) — да; данные о **выловах в тоннах** — нет, их нужно брать из других источников (см. [VYLOVY_SOURCES.md](VYLOVY_SOURCES.md)).

## 3. Рекомендации для проекта

| Задача | Источник | Действие |
|--------|----------|----------|
| Владельцы/операторы судов, контакты | **CRAVT** | На сайте CRAVT/IMCS API не описан. Уточнить наличие API или выгрузок: mcs.network@imcsnet.org. При появлении доступа — скрипт обогащения по IMO/названию (cravt_*). |
| Фото судов | **CRAVT** (если есть в API), **FleetPhoto** (уже есть) | FleetPhoto уже даёт фото для РМРС; CRAVT — дополнение для судов в реестрах RFMOs. |
| Периоды тралений, активность | **GFW** | Уже используем; при необходимости — Events API для детализации по судну. |
| Выловы в тоннах | **Не GFW, не CRAVT** | Росрыболовство, парсинг PDF/XLS с fish.gov.ru или запрос открытых данных (см. VYLOVY_SOURCES.md). |

Если у CRAVT появится API или выгрузка, можно добавить скрипт `scripts/enrich_vessels_from_cravt.py`: чтение `data/gfw_our_vessels.json`, поиск по IMO/названию, запись owner/operator и при наличии — фото/источники.
